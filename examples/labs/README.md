# Hands-On Labs

This directory contains structured lab exercises that combine concepts from multiple chapters.

## Available Labs

### Lab 1: First Deployment
Deploy your first application to Kubernetes with proper health checks and monitoring.
- **Prerequisites:** Chapters 1, 4, 15
- **Duration:** 45 minutes
- **Difficulty:** Beginner

### Lab 2: Infrastructure as Code
Build and manage cloud infrastructure with Terraform.
- **Prerequisites:** Chapters 6-9
- **Duration:** 90 minutes
- **Difficulty:** Intermediate

### Lab 3: Observability Stack
Set up a complete observability stack with metrics, logs, and traces.
- **Prerequisites:** Chapters 17, 19, 22
- **Duration:** 120 minutes
- **Difficulty:** Intermediate

### Lab 4: Secure CI/CD Pipeline
Build a secure CI/CD pipeline from scratch.
- **Prerequisites:** Chapters 23, 24, 28
- **Duration:** 90 minutes
- **Difficulty:** Advanced

### Lab 5: Production Readiness
Prepare an application for production with all best practices.
- **Prerequisites:** All parts I-VI
- **Duration:** 180 minutes
- **Difficulty:** Advanced

## How to Use Labs

1. Complete the prerequisite chapters first
2. Read the lab's README.md
3. Follow the step-by-step instructions
4. Experiment and break things (safely!)
5. Review the solution if you get stuck
6. Clean up resources when done

## Lab Structure

Each lab contains:
- `README.md` - Instructions and objectives
- `start/` - Starting point files
- `solution/` - Complete solution
- `test.sh` - Automated validation
- `cleanup.sh` - Resource cleanup

## Tips

- Don't peek at the solution too early
- Take notes as you go
- Try variations and experiments
- Share your learnings with others
- If stuck, review the relevant chapter
