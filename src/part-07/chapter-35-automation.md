# Chapter 35: The Automation Advocate

> *"Automate the toil, but don't automate away understanding."*

## What You'll Learn

Sarah spends hours each week on repetitive tasks that should be automated. By the end of this chapter, you'll know how to:

- Identify tasks worth automating vs. those better done manually
- Build reliable automation that handles edge cases and failures gracefully
- Use infrastructure as code and configuration management effectively
- Implement self-service tools that empower other teams
- Balance automation complexity with maintainability
- Measure the ROI of automation efforts and prioritize accordingly
